Bowler Scala Web Framework version 0.3 Released!

April 6th 2011
Version 0.3 of Bowler, a RESTful, multi-channel ready web framework in Scala has been released. For details on how to get the code, get started etc, please check the homepage or the manual.

This is the third release and has focused on cleaning up API's, better UI re-use and the addition of more Component oriented UI aspects with the help of Scalate's Squery.
The Bowler Example application in our GitHub repository showcases a lot of the new features.
A list of changes and new features below:

Change highlights for version 0.3

This is a partial changelist compiled from memory, hence it is likely to contain omissions.
Most of all, I am excited about adding Squery to the list of supported templating methods, a perfect choice for a more Component oriented style of templating. And, nothing stops you from mixing and matching as you see fit: maybe use the default Scalate methods for most things, but include/import use of Squery Components?

Feature Highlights

  • Scalate's designer friendly (Wicket like) Squery templating now supported, supports markup inheritance, component inheritance and composition.
  • Renderable has new "renderWith"-function that allows you to pass an alternate "ViewPath" - allows for better markup re-use between paths.
  • renderWith also allows rendering of Squery templates.
  • Re-usable ValidationFeedbackPanel Component for rendering validation errors (to save you some work if it is sufficient).
  • Optimized template loading, specifically for Squery templates.

API changes

  • removed TemplateRegistry.templateOverride - renderWith in Renderable with ViewPath now recommended method

Bugs fixed

  • CrudController paging bug fixed
  • Fixed dependency issue in not providing transient dependencies in all cases
