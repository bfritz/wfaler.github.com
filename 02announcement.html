Bowler Scala Web Framework version 0.2 Released!

February 24th 2011
Version 0.2 of Bowler, a RESTful, multi-channel ready web framework in Scala has been released. For details on how to get the code, get started etc, please check the homepage or the manual.

This is the second release and has focused on cleaning up API's and providing integration with JPA and Squeryl for persistence, as well as providing generic CRUD functionality for simple CRUD apps.
The Bowler Example application in our GitHub repository showcases a lot of the new features.
The full list of changes and new features below:

Changelist for version 0.2

This is a changelist compiled primarily by trawling through the commit logs since the 0.1 release, hence it is made with reservations for any potential omissions.

Features

  • ComponentRenderSupport for allowing simple creation of re-usable UI components.
  • ModelValidatorBuilder for allowing model objects to have default validators associated with them.
  • Updated to Scalate 1.4
  • Singleton validator registration allowed (useful for dynamically registering validators)
  • Persistence Mapper Modules for:
    • Squeryl
    • JPA
  • Persistence Mapper Modules include:
    • Transaction management with "Open Session in View" pattern (for single database apps)
    • UniqueEntityValidator validator
    • Generic EntityTransformer StringValueTransformer for looking up/mapping objects from id's.
    • Dao trait and AbstractDao to easily create mappers for other persistence frameworks
    • Generic "CrudController" that gives basic Create, Read, Update, Delete functionality in a controller for persistence mappers.

API changes

  • Scala "apply" for shorthand access replaces:
    • TransformerRegistry: resolveTransformer
    • GenericsParser now GenericTypeDefinition object with apply
    • AliasRegistry now has "apply" for alias lookups rather than "getAlias"
  • HTTP.Method changed from Enumeration to sealed trait of HttpMethod and case objects
  • Validators now mostly use companion objects for creation, eg "new MinimumLengthValidator" now constructed with "MinLength"
  • ContentTypeResolver object removed
  • signature of Request getAcceps changed and renamed to getAccept

Bugs fixed

  • rendering empty Seq/List bug fixed
  • Now allows rendering without a default layout
  • Improved error messages on missing messages for validators
